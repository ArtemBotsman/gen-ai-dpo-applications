# Постмортем · Инцидент MOB-2025-0314 · двойные списания

Дата инцидента: 14 марта 2025, 09:12–14:47 МСК.
Severity: P1. Затронуто: ~340 пользователей подписки «Сфера Плюс».

## Симптомы
Пользователи сообщают о двойном списании 1890 ₽ при автопродлении подписки.
Примеры обращений: Елена (interview_elena, тикет #88412), ещё 28 тикетов в первый час.

## Root cause
Race condition в billing-service: при таймауте ответа платёжного шлюза повторный retry
отправлял второй charge без idempotency key. Затронута версия API 2.3.1.

## Влияние на переводы
Параллельно 11 пользователей сообщили о «зависших» исходящих СБП-переводах на 2–8 минут.
Причина другая — перегрузка очереди после рестарта pod'ов billing-service.

## Timeline
09:12 — рост тикетов в поддержку
09:40 — эскалация on-call
11:00 — hotfix: отключение retry без idempotency
14:47 — все двойные списания идентифицированы, возврат инициирован
18:00 — 98% возвратов зачислены

## Коммуникация
Push и email затронутым пользователям. Текст согласован с compliance (ФЗ-152: не раскрывать
лишние ПДн в уведомлении).

## Action items
1. Обязательный idempotency key — до 21.03
2. Интеграционные тесты billing — 5 новых сценариев
3. Runbook для P1 в мобильном канале
## Подробный timeline и уроки

Утром 14 марта мониторинг показал всплеск ошибок billing-service. Параллельно в чат
пошли сообщения о двойном списании 1890 ₽ за подписку «Сфера Плюс». За первый час —
29 тикетов, среди них Елена из пользовательского интервью (тикет 88412).

Техническая причина: при таймауте платёжного шлюза retry уходил без idempotency key,
и часть пользователей получала второй charge. Затронута версия API 2.3.1. Отдельно
11 человек жаловались на зависшие исходящие СБП на 2–8 минут — это уже перегрузка
очереди после рестарта pod'ов, не тот же баг, но коммуницировали вместе.

К 11:00 выкатили hotfix — отключили опасный retry. К 14:47 список пострадавших
собран, возвраты запущены. К 18:00 деньги вернули 98% пользователей. Остальным —
на следующий рабочий день из-за лимитов шлюза.

Коммуникация согласована с юристами: в push не писали суммы и лишние персональные
данные — требование ФЗ-152. Текст нейтральный: «обнаружено дублирование платежа,
средства будут возвращены».

## Что делаем дальше

Обязательный idempotency key до 21 марта. Пять новых интеграционных тестов на billing.
Runbook для P1 в мобильном канале — чтобы не искать ответственных по памяти.
