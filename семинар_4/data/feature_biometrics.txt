# Спецификация фичи · Биометрический вход v2

Версия документа: 1.3. Автор: Команда Mobile Platform. Дата: 05.02.2025.

## Проблема
12% пользователей Android отключают биометрию после неудачных попыток. На Face ID
процент ниже — 4%. Жалобы в поддержку: «биометрия сбрасывается после обновления»,
«просит пароль каждый раз».

## Цели
- Снизить отказы биометрии до 8% на Android
- Сохранить соответствие ФЗ-152: биометрические шаблоны не покидают secure enclave
- Время входа < 1.2 с на p90

## Требования

### Функциональные
1. Повторная привязка Face ID / Touch ID без повторного KYC
2. Fallback на PIN при 3 неудачных попытках
3. Логирование только метаданных (успех/отказ), без шаблонов

### Нефункциональные
- Совместимость: iOS 15+, Android 10+
- При смене устройства — полная переавторизация (требование compliance_audit)

## Риски
- Samsung Knox: известные баги на A-серии — тестировать отдельно
- Пользователи путают биометрию входа в приложение и биометрию подтверждения платежа

## Метрики успеха
- Adoption биометрии: 72% → 80%
- Тикеты категории «биометрия / вход»: −30%

## Связанные документы
compliance_audit (хранение ПДн), support_q3_report (базовая линия тикетов).
## Контекст и пользовательские истории

История 1: пользователь Android после обновления 4.2 каждый раз вводит PIN — биометрия
«слетела». История 2: путает биометрию входа и подтверждения платежа — жмёт Face ID
дважды и злится. История 3: Samsung A54 — известные баги Knox, отдельный тест-план.

Цели релиза v2: снизить отказы биометрии на Android с 12% до 8%, ускорить вход
до 1.2 с на p90, не нарушить ФЗ-152 — шаблоны не покидают secure enclave на iOS
и Trusted Environment на Android. Юристы из compliance_audit проверили формулировки
согласия — ок с замечанием по дате политики.

Функционально: повторная привязка Face ID без KYC, fallback на PIN после трёх ошибок,
логируем только факт успеха/отказа. При смене телефона — полная переавторизация,
это требование безопасности, не баг.

Метрики успеха: adoption 72% → 80%, тикеты «биометрия / вход» −30% к базе из отчёта
поддержки Q3. Риски — путаница двух видов биометрии, баги Samsung. Релиз планируем
в апреле после пилота на 5% пользователей.


## Приложение А · Расширенные заметки полевой команды

Фрагмент A.1: Респондент R-07 (Москва, 38 лет) сравнивает мобильные клиенты банков:
«Т-Банк на личные траты, Сфера на ИП. СБП в Т-Банке быстрее, но тариф Бизнес-Плюс
окупается эквайрингом». Дублирует interview_ivan из другого канала рекрутинга.

Фрагмент A.2: 9 из 12 в топ-3 раздражителей назвали push-уведомления: «спам»,
«навязчивые кредиты», «не отключаются». Совпадает с support_q3_report и reviews_digest.

Фрагмент A.3: R-11 (Казань): «тормозит лента операций при фильтре по месяцу» — как
у interview_elena. На Android 13 cold start 3.1 с при слабом Wi-Fi.

Фрагмент A.4: Позитив про поддержку — возврат двойного списания за вечер, ответ
по СБП-лимиту за 15 минут. Согласуется с support_q3_report.

Фрагмент A.5: При обсуждении логов всплывал ФЗ-152 — отсылка к compliance_audit.
