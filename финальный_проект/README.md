# Лаборатория справедливости

Финальный проект курса «Практическое применение генеративного ИИ».  
Трек A — исследовательский (homo silicus): LLM-агенты как дешёвые испытуемые.

**Важно:** все участники эксперимента — синтетические персоны, сгенерированные моделью. Это не реальные люди и не результаты полевого опроса.

---

## О чём проект

Мы проверяем одну гипотезу: если дать языковой модели разные «социальные профили» (возраст, доход, город), она в целом повторит *логику* классической экономической игры «ультиматум» — малые предложения чаще отклоняются, — но *цифры* будут отличаться от тех, что получают у живых людей в лабораториях.

Чтобы это проверить, мы прогнали три связанных этапа на **одних и тех же шести персонах**:

1. **Фокус-группа** — персоны обсуждают между собой, как честно поделить 100 рублей между двумя незнакомцами. Получается текстовый транскрипт, как на настоящей групповой дискуссии.

2. **Анкета** — те же персоны отвечают на короткий опросник про доверие, риск и справедливость. Это silicon sampling: модель «играет роль» человека с заданным профилем. Ответы сравниваем с усреднёнными данными европейского опроса ESS.

3. **Игра «ультиматум»** — каждая персона сначала предлагает, сколько из 100 ₽ отдать второму, потом отвечает на фиксированные предложения (10, 20, 30, 40, 50 ₽): принять или отказать. Результаты сравниваем с опубликованными данными по реальным испытуемым.

## Почему Ghost 0/15 — и что это вообще значит

**Ghost-цитата** (призрачная цитата) — это когда модель при анализе текста *приписывает* участнику фразу, которой **не было** в исходном транскрипте. Классическая галлюцинация RAG/аналитики: «красивый» вывод с выдуманной подложкой.

Как это работает у нас:

1. Фокус-группа генерирует реальные реплики → они лежат в `output/focus_transcript.json`.

2. Отдельный шаг анализа просит модель для каждого участника выделить аспекты справедливости (равенство, альтруизм, наказание за жадность и т.д.) и **привести дословную цитату** из его реплик → результат в `output/transcript_aspects.json`.

3. Программа `analyze.py` **не доверяет модели на слово**: для каждой из 15 цитат она проверяет, есть ли эта фраза (или её фрагмент длиной от 8 символов) в исходном транскрипте.

4. Если цитаты в тексте нет — она попадает в список ghost. Если все 15 нашлись — получаем **0/15**.

**Почему именно ноль в нашем прогоне:**

- Модель на шаге анализа работала с температурой 0 (без «творчества») и в основном копировала куски реальных реплик, а не перефразировала их «от себя».
- Транскрипт и анализ делались одной и той же моделью в одной сессии — ей проще было цитировать то, что она сама же только что сгенерировала.
- Проверка достаточно мягкая: достаточно, чтобы цитата была **подстрокой** текста, а не совпадала буква в букву.

**Честная оговорка:** 0/15 не значит «галлюцинаций нет вообще». Это значит, что **конкретно при извлечении аспектов из фокус-группы** все 15 цитат нашлись в исходнике. Модель могла исказить смысл, числа в ultimatum или ответы анкеты — это другие виды ошибок, они видны в eval.

Результат проверки — в `output/ghost_quotes.json` (там `"ghost_quotes": 0`, `"total_quotes": 15`).

## Где посмотреть, что происходило и что показало

Все артефакты лежат в папке **`output/`**. Открывай их в Cursor, VS Code или любом просмотрщике JSON.

### Общая сводка прогона

**`output/run_summary.json`** — сколько раз дергали модель, сколько секунд заняла каждая фаза, итоговый ghost-rate и оценка judge.

**`output/pipeline_run.log`** — текстовый лог из терминала: что шло по шагам и финальная строка «Готово: 67 LLM-вызовов…».

### Фаза 0 — персоны

**`output/personas.json`** — шесть синтетических людей: имя, возраст, город, доход, биография и как каждый понимает «справедливость». Это «испытуемые» для всех следующих этапов.

### Фаза 1 — фокус-группа

**`output/focus_transcript.json`** — полный текст дискуссии: кто что сказал про делёж 100 ₽.  
Пример: почти все сошлись на «50 на 50 — базовая справедливость», но добавили идею добровольной помощи нуждающемуся.

### Анализ фокус-группы

**`output/transcript_aspects.json`** — по каждому участнику: какие темы (аспекты) нашла модель и **какие цитаты** к ним приложила.  
Сравни любую цитату отсюда с репликой того же человека в `focus_transcript.json` — увидишь, что они совпадают (отсюда и ghost 0/15).

**`output/ghost_quotes.json`** — итог проверки галлюцинаций в цитатах.

**`output/judge_report.json`** — LLM-as-judge: насколько обоснованы три утверждения («участники различаются», «есть тема наказания за жадность», «есть консенсус про 50/50»). Оценка judge: 0.7 из 1.

### Фаза 2 — анкета

**`output/survey_answers.json`** — ответы каждой персоны по шкале 1–5: доверие к людям, готовность к риску, минимально приемлемая доля в ultimatum и т.д.  
Eval сравнивает их с baseline из `input/ess_survey_baseline.json`.

### Фаза 3 — ultimatum

**`output/ultimatum_rounds.json`** — самое наглядное место для «что показало»:

- для каждой персоны строка `"role": "proposer"` — сколько ₽ она предложила бы отдать (например, p1 предложила 40);
- строки `"role": "responder"` — приняла ли персона офферы 10, 20, 30, 40, 50 ₽ и **почему** (поле `reasoning`).

Главный вывод: **responder-ы отказывают очень часто** — даже при 30–40 ₽, хотя у живых людей к этим суммам acceptance уже заметно выше. Отсюда общая доля принятий 20%.

### Итоговая оценка (eval)

**`output/eval_results.json`** — 55 тестовых кейсов с pass/fail:

- 30 кейсов — ultimatum responder (offer → accept vs human baseline);
- 6 — ultimatum proposer (offer в разумном диапазоне);
- 18 — анкета vs ESS;
- 1 — ghost-check.

У каждого кейса есть `"ok": true/false` и `"path"` — из какой фазы и персоны он взялся.

## Как устроен код (кратко)

| Файл | Зачем |
|------|-------|
| `pipeline.py` | запускает всё по порядку, складывает json в output/ |
| `personas.py` | генерирует персон по dem-профилям из input/ |
| `focus_group.py` | фокус-группа |
| `survey.py` | анкета |
| `ultimatum.py` | игра |
| `analyze.py` | аспекты, ghost-check, judge |
| `eval.py` | прогон метрик по готовым артефактам |
| `input/` | исходные профили и эталоны для сравнения с людьми |
| `отчёт.md` | текст для сдачи преподавателю |

Техники курса: синтетические персоны, structured output (Pydantic), field_validator, IE/аспекты, LLM-as-judge, проверка ghost-цитат, агенты в рынке (ultimatum).

## Как запустить

Установи зависимости, положи токен в `.env` (образец — `.env.example`):

```
pip install -r requirements.txt
python pipeline.py      # полный прогон (~67 вызовов LLM)
python eval.py          # оценка по output/
```

Для проверки структуры без API: `python pipeline.py --mock`

## Главные выводы (человеческим языком)

1. **Фокус-группа** показала мягкий консенсус: «поровну — база, но можно помочь нуждающемуся».

2. **Анкета** формально близка к ESS (18/18 в eval), но это слабый критерий — модель «ровно» отвечает в рамках шкалы.

3. **Ultimatum** — самое интересное: агенты **слишком principled**: отказывают даже при 30–40 ₽, тогда как люди в экспериментах чаще соглашаются. Гипотеза частично подтвердилась: направление верное, цифры — нет.

4. **Ghost 0/15** — при анализе фокус-группы модель не выдумала ни одной цитаты «из воздуха»; все 15 фраз реально есть в транскрипте.

Подробнее — в `отчёт.md`.
