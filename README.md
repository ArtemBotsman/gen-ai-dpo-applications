# Домашние работы — Практическое применение генеративного ИИ

Мои решения по курсу ИТМО. Стартовые материалы семинаров — в репозитории [paNikitin/gen-ai](https://github.com/paNikitin/gen-ai).

## Семинар 2 — заявки на ДПО

Папка `семинар_2/starter`. Генерация 50 синтетических заявок на курсы повышения квалификации: Pydantic-схема, стратификация по городам и специальностям (не больше 10% на каждое значение).

## Семинар 3 — анализ отзывов

Папка `семинар_3`. Пайплайн для отзывов на мобильное приложение: извлечение сущностей, аспекты, autodiscovery, Map-Reduce, judge, multi-doc.

## Семинар 4 — RAG

Папка `семинар_4`. Свой корпус про «Сфера Банк», gold-разметка, гибридный поиск, сравнение fixed и recursive чанкинга.

## Семинар 5 — макро-агент

Папка `семинар_5/starter`. ReAct-агент с данными ЦБ и Росстата, инструмент `compare_periods`, лог `trace.jsonl`, eval на 10 вопросов.

## Семинар 6 — Планировщик-Исполнитель-Критик

Папка `семинар_6/starter`. Мультиагентный паттерн PWC поверх агента из пятого семинара: валидатор схемы плана, параллельные workers, замер угодливости Критика, eval на 6 вопросах в трёх конфигурациях.

## Финальный проект — Лаборатория справедливости

Папка `финальный_проект`. Трек A (homo silicus): фокус-группа, анкета silicon sampling, игра ultimatum, проверка ghost-цитат. Запуск: `python pipeline.py`, затем `python eval.py`. Отчёт — `отчёт.md`.

## Как запускать

В каждой папке лежит `requirements.txt` — ставим зависимости, копируем `.env.example` в `.env` и прописываем токен LLM. Я работал через DeepSeek (`LLM_BASE_URL=https://api.deepseek.com`), но подойдёт любой OpenAI-совместимый API.

Файл `.env` в git не кладём.
