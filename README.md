# Лабораторная 2: заявки на курсы ДПО

Здесь лежит моя версия домашнего задания по семинару 2: нужно было сгенерировать
50 синтетических заявок на курсы повышения квалификации и проверить, что модель
не уходит в однотипные ответы.

Основная папка с решением:

`семинар_2/starter`

Что внутри:

- `schema.py` — схема заявки на Pydantic: ФИО, возраст, адрес, специальность,
  курс, стаж и год выпуска.
- `generator.py` — основной скрипт генерации. Он вызывает LLM через `make_client()`,
  просит ответ сразу под `Application` и использует `max_retries=3`.
- `applications.csv` — итоговые 50 заявок. Вложенный `address` распакован в
  обычные колонки `city` и `district`, чтобы файл было удобно проверять.
- `cities.png` и `specialities.png` — графики распределений.
- `выводы.md` — короткий разбор результата: где могли быть перекосы, как я с
  ними боролся и сработал ли валидатор.

Я выбрал вариант на «отлично» со стратификацией. Сначала задаю квоты по городам
и специальностям, а уже потом отдаю модели карточку конкретной заявки. Так меньше
шансов, что она начнёт постоянно выбирать Москву, Санкт-Петербург или одну и ту
же популярную профессию. В итоговом файле каждый город и каждая специальность
встречаются по 5 раз.

## Как запустить

```bash
cd "семинар_2/starter"
python -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
```

Дальше нужно передать настройки модели. Я не храню токен в коде, поэтому его
лучше положить в локальный `.env` рядом с `generator.py`:

```env
LLM_BASE_URL=https://api.deepseek.com
LLM_AUTH_TOKEN=твой_токен
LLM_MODEL=deepseek-v4-flash
```

`.env` не надо коммитить в репозиторий.

После этого можно запускать генератор:

```bash
python generator.py
```

Скрипт обновит `applications.csv`, `applications.json`, оба графика и
`выводы.md`.

## Что получилось

Текущий набор данных прошёл проверку:

- валидных заявок: `50/50`;
- максимальная доля одного города: `10%`;
- максимальная доля одной специальности: `10%`.

Пороги из задания выполняются с запасом: по городам нужно было не больше `40%`,
по специальностям — не больше `35%`.
